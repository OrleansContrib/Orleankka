    Register(String, TimeSpan, Boolean, Boolean, Object)

    Registers a one-off timer to a due time to send Timer message to this actor Receive(Object) function.

    Declaration
    void Register(string id, TimeSpan due, bool interleave = true, bool ignore = false, object message = null)
    Parameters
    Type Name Description
    String id

    Unique id of the timer
    TimeSpan due

    Due time for sending timer message.
    Boolean interleave

    Controls timer message interleaving. Set to false for non-interleaved delivery
    Boolean ignore

    If set to true the message will be sent in fire-and-forget fashion
    Object message

    Custom message to send on timer tick
    Remarks

    This timer will not prevent the current grain from being deactivated. If the grain is deactivated, then the timer will be discarded.

    Until the Task returned from the Receive(Object) is resolved, the next timer tick will not be scheduled. That is to say, timer callbacks never interleave their turns.

    Any exceptions thrown by or faulted Task's will be logged unless the ignore is set to true

    For non-interleaved delivery set interleave to false. The message will be delivered via Self method. If used in conjunction with ignore set to false the message will be delivered as OneWay for most optimal performance

    Use message parameter to send custom message

    Register(String, TimeSpan, TimeSpan, Boolean, Boolean, Object)

    Registers a timer to periodically send Timer message to a given actor Receive(Object) function.

    Declaration
    void Register(string id, TimeSpan due, TimeSpan period, bool interleave = true, bool fireAndForget = false, object message = null)
    Parameters
    Type Name Description
    String id

    Unique id of the timer
    TimeSpan due

    Due time for sending timer message.
    TimeSpan period

    Period of subsequent timer ticks.
    Boolean interleave

    Controls timer message interleaving. Set to false for non-interleaved delivery
    Boolean fireAndForget

    If set to true the message will be sent in fire-and-forget fashion
    Object message

    Custom message to send on timer tick
    Remarks

    This timer will not prevent the current grain from being deactivated. If the grain is deactivated, then the timer will be discarded.

    Until the Task returned from the Receive(Object) is resolved, the next timer tick will not be scheduled. That is to say, timer callbacks never interleave their own turns.

    NOTE: If fireAndForget is set to true the next timer tick will be scheduled immediately.

    The timer may be stopped at any time by calling the Unregister(String) method

    Any exceptions thrown by or faulted Task's returned from the Receive(Object) will be logged (unless the fireAndForget is set to true), but will not prevent the next timer tick from being queued.

    For non-interleaved delivery set interleave to false. The message will be delivered via Self method. If used in conjunction with fireAndForget set to false the message will be delivered as OneWay for most optimal performance

    Use message parameter to send custom message
